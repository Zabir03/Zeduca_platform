3. Keys & Constraints

  • Primary Key: unique identifier.
  • Candidate Key: minimal unique attribute set.
  • Foreign Key: enforces referential integrity to other tableâ€™s PK.
  • Constraints: Unique, Not Null, Check, Default.

Example SQL:

CREATE TABLE Student (
  sid INT PRIMARY KEY,
  name VARCHAR(100) NOT NULL
);

CREATE TABLE Enroll (
  sid INT,
  cid INT,
  grade CHAR(2),
  PRIMARY KEY (sid, cid),
  FOREIGN KEY (sid) REFERENCES Student(sid)
);

5. SQL â€” DDL / DML / DCL / TCL

  • DDL: CREATE, ALTER, DROP
  • DML: INSERT, UPDATE, DELETE
  • DQL: SELECT
  • TCL: COMMIT, ROLLBACK, SAVEPOINT
  • DCL: GRANT, REVOKE

Joins:

-- Inner join

Group / Having / Aggregate:

FROM Enroll
GROUP BY cid
HAVING COUNT(*) > 10;

Window functions:

SELECT sid, cid, grade,
       RANK() OVER (PARTITION BY cid ORDER BY grade DESC) AS rank
FROM Enroll;

6. Normalization

Goal: remove redundancy, avoid anomalies (insert/update/delete).

Normal forms:

  • 1NF: atomic attributes (no repeating groups).
  • 2NF: 1NF + no partial dependency on part of composite PK.
  • 3NF: 2NF + no transitive dependency.
  • BCNF: every determinant is a candidate key.

7. Transactions, ACID & Isolation

Transaction: atomic unit of DB work.

ACID Properties:

  • Atomicity: all or nothing.
  • Consistency: DB moves between valid states.
  • Isolation: transactions appear isolated.
  • Durability: committed changes survive crash.

Isolation Levels:

  • READ UNCOMMITTED
  • READ COMMITTED
  • REPEATABLE READ
  • SERIALIZABLE

9. Indexes

Why: speed up lookups, sorting, range queries. Tradeoff: extra storage + slower writes/updates.

Types:

  • B+-Tree index: good for range queries. Most RDBMS use B+ trees.
  • Hash index: O(1) equality lookups, not good for range queries.
  • Clustered index: table stored in index order (one per table).
  • Non-clustered: separate index structure with pointers to data rows.

14. NoSQL, CAP theorem, BASE

NoSQL types:

  • Key-Value (Redis): super fast for simple retrieval.
  • Document (MongoDB): JSON-like flexible schema.
  • Column-family (Cassandra): wide rows, good for time-series.
  • Graph (Neo4j): relationships-first queries.

CAP Theorem:

In a partitioned system you can guarantee only two of: Consistency, Availability, Partition-tolerance.

BASE:

Basically Available, Soft state, Eventual consistency â€” looser than strict ACID for higher availability.
