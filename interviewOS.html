The Complete Guide to Operating Systems

Core concepts for interviews and beyond.

1. Introduction to OS

Definition:

An OS is a system software that manages hardware and software resources, and provides services to applications and users.

Functions:

  • Process management
  • Memory management
  • File management
  • I/O management
  • Security & Protection
  • User interface (CLI, GUI)

2. Types of Operating Systems

  • Batch OS: executes jobs in batches (no interaction).
  • Multiprogramming OS: multiple programs in memory.
  • Multitasking OS: multiple tasks at once (Windows, Linux).
  • Time-sharing OS: CPU time divided among users.
  • Distributed OS: multiple systems work together.
  • Real-time OS: strict deadlines (aircraft, robotics).

4. CPU Scheduling

Decides which process runs on CPU.

Scheduling Algorithms:

  • FCFS (First Come First Serve): Non-preemptive, simple. Drawback: Convoy effect.
  • SJF (Shortest Job First): Optimal but needs burst-time prediction.
  • Round Robin (RR): Preemptive, time slice.
  • Priority Scheduling: Starvation possible â†’ Aging used.
  • Multilevel Queue & Multilevel Feedback Queue

7. Memory Management

  • Logical vs Physical Address
  • Swapping
  • Contiguous Allocation: Fixed & Dynamic Partitioning
  • Fragmentation: Internal (unused space inside partition), External (free memory scattered).
  • Paging: Divide memory into frames & process into pages. Uses page table.
  • Segmentation: Memory divided into logical segments (code, data, stack).
  • Virtual Memory: Demand Paging (load pages when needed).
  • Page Replacement Algorithms: FIFO, LRU (Least Recently Used), Optimal (theoretical best).
